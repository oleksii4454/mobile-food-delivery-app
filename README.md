# Мобільний застосунок доставки їжі
Створено для лабораторного практикуму з технологій створення програмних продуктів

## Зміст
Цей репозиторій містить повний стек мобільного додатку для доставки їжі, що складається з двох основних частин:
1. **Frontend (`/frontend`)** - кросплатформний мобільний та веб-додаток, написаний на **Flutter**.
2. **Backend (`/backend`)** - серверна частина на **Node.js** (Express), **Prisma ORM** та база даних **PostgreSQL**.

## Передумови
Для запуску проєкту потрібні наступні інструменти
* **Docker** & **Docker Compose** (для серверної частини та бази даних)
* **Flutter SDK** (для запуску клієнтської частини)

## Запуск серверної частини
Вся інфраструктура сервера повністю контейнеризована, тому встановлювати Node.js чи PostgreSQL локально не потрібно.

### Налаштування змінних оточення
Створіть файл `.env` всередині папки `/backend`. Він має містити:
```env
PORT=3000
DATABASE_URL="postgresql://admin:admin@my_postgres_container:5432/delivery_db?schema=public"
JWT_SECRET=test
```

### Збір та запуск контейнерів
У кореневій директорії:
```shell
docker-compose up --build
```

### Міграція баз даних
Після запуску серверу:
```shell
docker-compose exec backend_api npx prisma migrate dev --name init
```

## Запуск клієнтської частини
Для завантаження залежностей:

```shell
cd frontend
flutter pub get
```

### Запуск додатку
```shell
flutter run
```
